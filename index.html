Agentic AI for consumers

An assistant that acts on the world—
not just one that reads it.

Most consumer AI today is an advanced search tool: it automates information-gathering to answer questions — the “read” tasks of deep research. flawed-flowed is different. With the right setup, it acts as a personal assistant that actually does things for you — booking, messaging, sharing, exploring — the “write” tasks that change the world on your behalf.

Browser-use as a universal primitive · Real tasks, end to end · 4 live demos below
See the demos ↓

Incumbents like Anthropic, OpenAI, and Google DeepMind are capitalizing on agentic AI in coding and enterprise. But there’s a large, largely untapped vertical: consumer-facing agentic AI. Many everyday tasks can’t be solved by reading the internet alone — the model has to write, using the web to actually book a table, message a business, or share a recommendation with a friend.

The four demos below each show flawed-flowed completing one of those write-tasks, end to end.

What’s still to do

Realizing this requires core advances on both the modeling and infrastructure sides.

Robustness & reliability

Unlike “superintelligence” tasks where the expectation is failure, being superhuman at everyday tasks means clearing a different bar: the expectation is success. The assistant has to perform at the level of a competent human executive assistant.

Efficiency

Once tasks are done reliably, they should be done quickly — for a better experience and lower cost. Yet users may value reliability over raw speed: the real win is shifting cognitive load, so a task done within a reasonable window — and trusted — can be enough.

There’s a strong correspondence between the cost of failure in a task and the rate of progress on it. We moved fast on math and code because checking correctness was cheap and failure was nearly free. Everyday write-tasks raise that cost — placing the wrong order isn’t free — which is exactly why reliability matters most here.

The approach

Simplicity and generality. Just as execution is shifting from tool-calling MCPs to terminal-only code execution, we build on simple, generalizable primitives rather than site-specific APIs.

Instead of bespoke tools for WhatsApp, Instagram, or any one service, we start from browser-use as a universal primitive — a basis broad enough to capture anything a human can do on the web, with no per-site integration. Today that’s a thin wrapper around Playwright (via playwright-cli), which post-processes the DOM to make it more readable for the model. As coding models improve, they can build their own robust APIs on top.
